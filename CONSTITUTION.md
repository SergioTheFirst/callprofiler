# CONSTITUTION.md — Принципы разработки CallProfiler

Этот документ устанавливает **непреложные правила** для всей команды
(разработчиков, AI-агентов, ревьюеров). Каждый коммит должен им соответствовать.

---

## I. Надёжность превыше скорости

1. **Никогда не роняем пайплайн.** Любая ошибка на одном файле логируется,
   статус в БД выставляется `error`, обработка продолжается со следующего.
2. **Идемпотентность.** Повторный запуск на уже обработанном файле даёт тот же
   результат, не создаёт дублей (проверка по MD5).
3. **Graceful degradation.** Если LLM недоступен — транскрипция сохраняется.
   Если pyannote упала — Whisper-сегменты сохраняются без ролей (`UNKNOWN`).

---

## II. Безопасность данных пользователя

4. **user_id-изоляция.** Каждый SQL-запрос, работающий с данными пользователя,
   ОБЯЗАН фильтровать по `user_id`. Запрещено выбирать "все записи" без фильтра.
5. **Без утечки токенов.** `hf_token` и `telegram_token` — только из конфига,
   никогда не хардкодить в коде.
6. **Конфиг не в репозитории.** `configs/base.yaml` с реальными токенами —
   в `.gitignore`. В репо только шаблон с `"TOKEN"`.

---

## III. Управление GPU-памятью

7. **Load → Use → Unload.** Whisper и pyannote выгружаются сразу после
   использования (`del model`, `gc.collect()`, `torch.cuda.empty_cache()`).
   RTX 3060 12GB не резиновый.
8. **Batch-режим.** Предпочтительный паттерн: загрузить все модели → обработать
   пакет → выгрузить. Не загружать/выгружать на каждый файл.
9. **Patch torch.load обязателен.** weights_only=False для pyannote 3.3.2.

---

## IV. Качество кода

10. **Один модуль — одна ответственность.** `normalizer.py` нормализует аудио,
    `whisper_runner.py` транскрибирует, `repository.py` хранит данные.
    Пересечений нет.
11. **Тесты для парсеров и репозитория.** `filename_parser.py` и
    `repository.py` — 100% покрытие публичного API тестами.
12. **UTF-8 везде.** Все файлы, все строки, все открытия файлов — `encoding="utf-8"`.
13. **Логировать, не печатать.** `logger = logging.getLogger(__name__)`,
    никаких `print()` в продакшн-коде (допустимо только в CLI).

---

## V. Совместимость стека

14. **Зафиксированный стек:** torch 2.6.0+cu124, faster-whisper, pyannote 3.3.2.
    Не обновлять без явного решения команды.
15. **use_auth_token=** (не `token=`) для pyannote 3.3.2 — API изменился
    в более поздних версиях.
16. **Без venv.** Системный Python, `pip install --break-system-packages`.
17. **Без Docker, Redis, WhisperX.** Только то, что в плане.

---

## VI. Git-дисциплина

18. **Осмысленные коммиты.** Один коммит — одна логическая единица изменений.
    Формат: `тип(модуль): описание` (feat, fix, refactor, docs, test).
19. **CHANGELOG.md актуален.** Каждый значимый коммит отражается в CHANGELOG.
20. **CONTINUITY.md актуален.** После каждой сессии обновляется статус и
    следующий шаг — чтобы любой агент/разработчик мог подхватить работу.
