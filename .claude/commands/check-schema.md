---
description: Проверить реальную схему БД перед написанием SQL (предотвращает баги)
---

Перед написанием любого SQL-запроса выполни:

1. Прочитай `src/callprofiler/db/schema.sql` — это source of truth для схемы.
2. Если БД существует (`data/db/callprofiler.db`), покажи фактическую схему через:
   ```
   python -m callprofiler inspect-schema
   ```

Затем подтверди пользователю список таблиц и колонок.
Только ПОСЛЕ этого пиши SQL.

**Почему это важно:**
- Имена колонок часто не совпадают с ожидаемыми (raw_response vs raw_llm_response).
- Изменения схемы могли не доехать до схемы документации.
- Миграции могли добавить/удалить колонки в БД.

Экономит итерации: лучше прочитать схему 1 раз, чем писать 3 неправильных SQL-запроса.
