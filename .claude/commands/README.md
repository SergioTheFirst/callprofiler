# Slash Commands для CallProfiler

Эти команды оптимизируют рутинные операции и экономят токены.

## Команды

| Команда | Когда использовать | Что делает |
|---------|--------------------|------------|
| `/brief` | В начале сессии | Читает CONTINUITY.md (100 строк) + CHANGELOG.md (40 строк), выдаёт "Last state / Next" |
| `/quick-status` | В любой момент | git status + последние коммиты + кол-во тестов, без чтения больших файлов |
| `/save` | В конце сессии | Tests → journal check → commit → push. Безопасная последовательность |
| `/check-schema` | Перед SQL-запросами | Читает schema.sql + выводит реальную схему БД |

## Зачем это нужно (экономия токенов)

**Без slash-команд:**
```
User: "Начни сессию, прочитай всё нужное и скажи статус"
Claude: [читает CONTINUITY.md целиком = 200+ строк]
        [читает CHANGELOG.md целиком = 500+ строк]
        [читает memory/*.md = 400+ строк]
        [выдаёт длинный summary]
```
Итог: ~2000 токенов на чтение, ~500 на вывод.

**Со slash-командой `/brief`:**
```
User: /brief
Claude: [читает только релевантные секции = 140 строк]
        [выдаёт ОДНУ строку "Last state / Next"]
```
Итог: ~400 токенов на чтение, ~50 на вывод. **Экономия 80%**.

## Как работают slash-команды

Каждый файл `.claude/commands/<name>.md` становится командой `/<name>`.
Содержимое файла = инструкция для Claude (не для пользователя).

YAML frontmatter `description` виден в автодополнении.

## Расширение

Можно добавлять свои команды по мере необходимости:
- `/test-single <file>` — запустить один тест-файл
- `/phase-advance` — перейти к следующей задаче roadmap
- `/emergency-save` — быстрый коммит без тестов (dangerous)

**Правило:** команда хорошая, если её описание умещается в 1 строку.
Если инструкция сложная — это не команда, а процесс.
