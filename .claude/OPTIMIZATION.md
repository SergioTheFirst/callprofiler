# Claude Code Optimization — что и почему

Этот файл описывает конфигурационные файлы `.claude/` и `.claudeignore`.
Применены рекомендации из статьи [vc.ru/ai/2868238](https://vc.ru/ai/2868238-nastroika-claude-code-dlya-optimizatsii-rabotyi-i-ekonomii-tokenov).

## Что применено

### 1. `.claudeignore` — экономия 30-40% токенов

Исключает из контекста Claude всё, что не нужно для работы с кодом:
- Python артефакты (`__pycache__/`, `*.pyc`, `.pytest_cache/`)
- Данные пользователя (`data/db/`, `data/logs/`, `*.db`)
- Медиафайлы (`*.mp3`, `*.wav`, `*.m4a`)
- `.git/` — огромная папка, не нужна для кода
- Исторические документы (`ARCHITECTURE_v3.md`, `reference_batch_asr.py` и т.д.)

**Эффект:** при старте сессии Claude не тратит токены на чтение БД/логов/кэшей.
**Правило:** добавляем только то, что 100% не нужно. В сомнении — не добавляем.

### 2. `.claude/settings.json` — проектные настройки

```json
{
  "env": {
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "60",
    "CLAUDE_CODE_SUBAGENT_MODEL": "haiku"
  }
}
```

**`CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=60`:**
- Компакция контекста на 60% заполнения вместо 80%
- Качество генерации падает уже на 20-40% заполнения (размытие внимания)
- 60% — компромисс между стабильностью и сохранением контекста

**`CLAUDE_CODE_SUBAGENT_MODEL=haiku`:**
- Субагенты (Agent tool) работают на Haiku вместо Sonnet
- Экономия 50-70% на исследовательских задачах
- Качество для read-heavy задач (поиск в коде, обзор структуры) достаточное

### 3. `.claude/commands/` — slash-команды

Переиспользуемые команды для рутины (см. `.claude/commands/README.md`):
- `/brief` — начало сессии (~80% экономии vs ручное чтение)
- `/quick-status` — компактный статус
- `/save` — безопасное сохранение
- `/check-schema` — проверка схемы БД перед SQL

### 4. `.claude/settings.local.json` — personal permissions

Безопасные разрешения (git, pytest, python) без подтверждения.
Личный файл, не коммитится в некоторых проектах. У нас закоммичен с 2026-03-30.

## Что НЕ применено (по решению пользователя)

- ❌ **Рефакторинг CLAUDE.md до 80 строк** — файл большой (700+ строк, исторический план),
  но содержит ШАГ 0-15 с детальной разработкой. Риск сломать > выгоды от экономии.
- ❌ **Hook блокировки правок на main** — пользователь разрешил прямой push в main.
- ❌ **Path-scoped rules (`.claude/rules/`)** — можно добавить при росте проекта.

## Измеримые эффекты

| Изменение | Ожидаемая экономия | Риск |
|-----------|-------------------|------|
| `.claudeignore` | 30-40% токенов/сессия | 0 (только read scope) |
| Haiku subagents | 50-70% на исследованиях | низкий |
| Autocompact 60% | меньше деградации | 0 |
| Slash commands | 80% на рутине | 0 |
